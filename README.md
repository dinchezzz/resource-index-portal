# resource-index-portal

## 项目简介

resource-index-portal 是一个用于归档和发布独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为 HTML 资源页面的集中存放与索引入口。

主要用途包括但不限于：

- 存放静态 HTML 页面归档
- 提供轻量级资源索引
- 便于通过 GitHub Pages 或其他静态托管服务直接访问

## 目录结构说明

```
resource-index-portal/
├── index.html          # 主入口页面，列出所有归档页面链接
├── pages/              # 存放各独立 HTML 页面
│   ├── example-page-1.html
│   ├── example-page-2.html
│   └── ...
├── assets/             # 公共资源文件（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `index.html`：仓库的入口页面，建议在此处提供所有归档页面的导航或索引。
- `pages/`：所有独立 HTML 页面存放于此目录，文件名应具有描述性。
- `assets/`：存放页面所需的公共资源文件，按类型分目录组织。

## 页面归档说明

本仓库中的每个 HTML 页面均为独立归档，不依赖外部域名或特定网络环境。页面内容可包含文本、图片、样式和脚本，但所有资源应尽量内联或存放于 `assets/` 目录下，以保证离线或静态托管时正常显示。

归档页面命名建议：

- 使用小写字母、数字和连字符
- 命名应体现页面主题或用途
- 避免使用空格或特殊字符

## 维护说明

- 添加新页面时，请将 HTML 文件放入 `pages/` 目录，并在 `index.html` 中添加对应链接。
- 更新已有页面时，请保持文件名不变，避免破坏已有链接。
- 如需删除页面，请同时移除 `index.html` 中的对应链接。
- 建议在提交说明中简要描述新增或修改的页面内容。

## 使用方式

可直接通过 GitHub Pages 或任何静态文件服务器访问本仓库的 `index.html` 文件，获取所有归档页面的索引。也可单独访问 `pages/` 目录下的任一 HTML 文件。

## 许可

本仓库内容按需使用，无特定许可证约束。如引用第三方资源，请遵守相应许可协议。
